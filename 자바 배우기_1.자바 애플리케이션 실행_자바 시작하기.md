# 자바 시작하기  

## 자바 애플리케이션의 요소  

편집기에 **일부 코드를 입력**하고 실행하여 첫 번째 Java 응용 프로그램이 실행되는 것을 보고 싶어한다는 것을 이해합니다! 걱정하지 마십시오. 이 튜토리얼이 끝나면 기대가 충족될 것이다. 그러나 계속 진행하기 전에 **수행 중인 작업을 완전히 이해하기 위해 알아야 할 몇 가지 요소**를 살펴보고자 한다.

다른 프로그래밍 언어에 익숙하더라도 **컴파일**에 대해 알고 **실행 파일**이 무엇인지 알고 **Java는 C 또는 C++와 다른 방식으로 작동**하기 때문에 다음에 관심이 있을 수 있다.  

<br/>

## 자바 코드 컴파일 및 실행  
* 텍스트 편집기에서 일부 Java 코드를 작성하는 것
* 즉, 컴파일러. 이 코드는 컴퓨터에서 실행할 수 있는 다른 형식으로 변환되어야 한다.  
* 컴파일 및 실행에는 JDK(Java Development Kit)의 일부인 두 가지 특정 소프트웨어가 필요하다.  

바이너리 또는 실행 파일은 사람이 아닌 컴퓨터만 읽을 수 있습니다. 바이트 코드(특수 이진 코드)가 포함되어 있기 때문이다.  

<br/>

## 첫 번째 자바 클래스 만들기  

첫 번째 단계는 작성 중인 Java 코드가 일반 텍스트 파일에 저장된다는 것이다.  
Java는 **객체 지향 언어**이다. 기억해야 할 점은 작성하는 모든 코드가 **Java 클래스**에 있어야 한다는 것이다.  
```java
public class MyFirstClass {
    
}
```
클래스의 이름은 MyFirstClass. 이 텍스트를 **MyFirstClass.java**에 저장해야 한다.  
확장자가 가진 클래스 = 파일 이름  
확장자: .java  
Java 규칙: 클래스의 이름은 대문자로 시작한다.  

<br/>

## 첫 번째 클래스 편집 준비  

두 번째 단계: MyFirstClass.java를 실행할 수 있는 다른 형식으로 변환하는 작업. -> MyFirstClass.class  

텍스트를 컴파일 하려면 **compiler(컴파일러)**가 필요하다.  

Java 다운로드 = JDK(Java Development Kit) 다운로드  

Java 응용 프로그램을 컴파일하고 실행하는 데 사용할 도구가 있다.  

JRE(Java Runtime Environment) - JDK의 하위 집합  

<br/>

## JDK(Java Developmet Kit) 셋팅  

* Linux/AArch64
* Linux/x64
* macOS/x64
* Windows/x64  

<br/>

### Windows/x64용 JDK 설정
**환경변수 설정**
```java
> set JAVA_HOME=D:\jdk\jdk-16
```
```java
> echo %JAVA_HOME%
```
```java
D:\jdk\jdk-16
```
JDK 디렉토리의 디렉토리 PATH를 추가하기 위해 환경변수를 업데이트 해야 한다.
```java
> set PATH=%JAVA_HOME%\bin;%PATH%
```

<br/>

### macOS용 JDK 설정  
```java
$ tar xzf *.tar.gz
```
.tar.gz: 현재 디렉토리에 있는 확장자를 가진 모든 파일을 확장한다.  
이 디렉토리의 확장자는 .jdk

JAVA_HOME이 작업이 완료되면 JDK를 확장한 디렉토리를 가리키는 환경 변수를 만들어야 한다.  
JDK 16 아카이브 파일을 확장하는 경우
```java
$ export JAVA_HOME=/Users/javauser/jdk/jdk-16.jdk/Contents/Home
```  
올바르게 변수가 설정되었는지 확인하기
```java
$ echo $JAVA_HOME
```
위 명령은 다음을 인쇄한다.
```java
/Users/javauser/jdk/jdk-16.jdk/Contents/Home
```  
JDK 디렉토리의 디렉토리 PATH를 추가하기 위해 환경변수를 업데이트 해야 한다.
```java
$ export PATH=$JAVA_HOME/bin:$PATH
```
모든 것이 정상인지 확인하기
```java
$ which java
```
위 명령은 다음을 인쇄한다.
```java
/Users/javauser/jdk/jdk-16.jdk/Contents/Home/bin/java
```

## 첫 번째 클래스 컴파일  

### 첫 번째 Java 프로그램 컴파일 및 실행
1. MyFirstClass.java로 첫 번째 수업을 저장한 디렉토리로 변경한다.
2. 다음을 입려갛여 컴퍼일러 액세스 할 수 있는지 확인하기  
```java
> java -version
```
현재 사용 중인 javac 버전을 알려 주어야 한다.  
오류 메시지 표시될 시 JAVA_HOME 변수를 확인한다.  

3. 이제 첫 번째 코드를 컴파일할 준비가 되었다. 다음을 입력한다.
```java
> javac MyFirstClass.java
```  

<br/>

## Hello World 프로그램을 단일 파일 응용 프로그램으로 실행  
Java SE 11부터는 프로그램이 단일 파일로 작성되어 있으면 컴파일 단계를 거치지 않고 Java 애플리케이션을 실행할 수 있다.   
이것이 간단한 Hello, World의 경우이다.  
```java
> java MyFirstClass.java
```  
Hello, World가 인쇄된다. 콘솔에 메시지가 표시된다.
